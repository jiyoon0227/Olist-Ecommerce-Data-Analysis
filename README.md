## 🛒 Brazilian E-Commerce (Olist) Data Analysis Project
"매출 성장의 이면, 물류 효율성이 고객 만족도에 미치는 임계점(Threshold) 분석"

### 1.  프로젝트 개요
브라질 최대 이커머스 플랫폼인 Olist의 10만 건 결제 데이터를 활용하여, 단순한 매출 현황 파악을 넘어 배송 성능이 비즈니스 지표(리뷰 점수)에 미치는 실질적인 영향을 정량적으로 분석한 프로젝트입니다.

### 2. 🛠 사용 기술 및 도구
Database: MySQL (Workbench)

Data Engineering: 기초 데이터 정제, 참조 무결성 검증, 분석용 데이터 마트(View) 구축

Analysis: SQL (Window Functions, Aggregation, Join Validation)

### 3. 📊 핵심 분석 결과 (Main Insights)
1️⃣ 매출 성장성 및 패턴 식별
블랙 프라이데이 효과: 2017년 11월, 전월 대비 **52.77%**의 폭발적 매출 성장 확인.

성장 동인 분석: 매출 급증은 객단가(AOV) 상승이 아닌 **주문 건수의 폭증(Volume-up)**에 기인함을 증명.

2️⃣ 물류 병목 현상과 리스크 관리
물류 과부하 포착: 매출 피크 시즌(11월)에 배송 지연율이 평시 대비 3배(16.16%) 급증하는 병목 구간 식별.

지역별 격차: 물류 허브(MG) 대비 북부 외곽 지역(AP, RR)의 배송 기간이 2.3배 길어지는 인프라 불균형 확인.

3️⃣ [핵심] 고객 만족도의 '마지노선' 발견
심리적 임계점: 배송 기간이 21일을 초과하는 순간, 평균 리뷰 점수가 4.0점에서 3.0점으로 급락(25% 하락).

신뢰의 가치: 절대적 배송 속도보다 **'예정일 준수 여부(Punctuality)'**가 만족도에 더 치명적임. (3일 이상 지연 시 평점 1.85점 기록)

### 4. 📂 프로젝트 구조 (SQL Roadmap)
01_database_setup.sql: 분석 환경 초기화

02_data_validation.sql: 원본 데이터의 무결성 및 이상치 11개 항목 검증

03_data_cleaning.sql: 데이터 타입 변환 및 노이즈 제거

04_join_validation.sql: 테이블 간 관계 및 데이터 팽창 위험 검증

05_business_analysis.sql: 핵심 KPI 도출 및 비즈니스 인사이트 요약

### 5. 🗝️ 기대 효과 및 제안
Targeting: 매출 비중 80%를 차지하는 상위 17개 카테고리에 마케팅 역량 집중 제안.

Operation: 고객 만족도 방어를 위해 배송 예정일 산출 알고리즘 고도화 및 21일 초과 배송 건에 대한 선제적 관리 필요성 역설.
